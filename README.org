A [[https://www.gnu.org/software/guile/][GNU Guile 3]] client for the [[https://hegel.dev][Hegel]] universal property-based testing protocol.

Hegel separates the testing engine ([[https://hypothesis.readthedocs.io/][Hypothesis]], running as =hegel-core=) from the language client. This library implements the HEGL wire protocol so you can write property-based tests in Guile while Hypothesis handles generation, shrinking, and search.

hegel-guile version tracks the Hegel protocol version: =0.7.x= for
protocol 0.7. Patch bumps (0.7.1 -> 0.7.2) are hegel-guile fixes.
A protocol bump from hegel-core (0.7 -> 0.8) triggers a full
spec/compliance rebuild and a minor version bump here.

** Requirements

- GNU Guile 3.x
- Python 3.11+ with [[https://pypi.org/project/hegel-core/][hegel-core]] (for the test server)

** Quick Start

#+begin_src sh
# Install hegel-core
uv tool install hegel-core

# Run unit tests (no server needed)
gmake test

# Run property tests against live server
guile3 -L src examples/basic.scm
#+end_src

** Usage

#+begin_src scheme
(use-modules (hegel))

(define-hegel-test (my-test tc #:test-cases 100)
  (let ((x (tc-draw tc (integers #:min-value 0 #:max-value 1000)))
        (y (tc-draw tc (integers #:min-value 0 #:max-value 1000))))
    (unless (= (+ x y) (+ y x))
      (error "addition not commutative" x y))))

(run-hegel-tests!)
#+end_src

| =floats=       | float            | =#:min-value=, =#:max-value=, =#:allow-nan=, =#:allow-infinity=  |
| =text=         | string           | =#:min-size=, =#:max-size=                                       |
| =binary=       | binary           | =#:min-size=, =#:max-size=                                       |
| =null-values=  | null             | ---                                                              |
| =lists-of=     | list             | =#:min-size=, =#:max-size=, =#:unique=                           |
| =one-of=       | (top-level key)  | variadic schemas                                                 |
| =sampled-from= | (top-level key)  | list of items                                                    |
| =const-value=  | (top-level key)  | single value                                                     |
| =gen-filter=   | (client-side)    | predicate, uses =tc-assume= for rejection                       |
| =gen-map=      | (client-side)    | transform function                                               |
