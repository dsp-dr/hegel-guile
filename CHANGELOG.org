* [0.7.1] - 2026-03-27

Initial release. Implements the HEGL wire protocol for hegel-core 0.2.3.

** Added

*** Wire Protocol Stack
- CBOR codec (=cbor.scm=) --- RFC 8949 subset covering major types 0--5 and 7
- CRC32 checksum (=crc32.scm=) --- pure Scheme, no C dependencies
- HEGL packet framing (=packet.scm=) --- magic =0x4845474C=, CRC32 integrity, channel multiplexing
- Packet demultiplexer (=mux.scm=) --- per-channel buffering for interleaved server traffic
- Channel abstraction (=channel.scm=) --- message ID sequencing, reply validation, ={result: v}= envelope unwrap

*** Protocol Layer
- Protocol message constructors (=protocol.scm=) --- =run_test=, =generate=, =assume=, =mark_complete= matching hegel-core 0.2.3 vocabulary
- Raw bytes handshake --- ="hegel_handshake_start"= / ="Hegel/0.7"= with version validation
- Server-driven test case lifecycle (C14) --- event loop reads =test_case= / =test_done= events from server
- Server manager (=server.scm=) --- spawn hegel-core subprocess, socket discovery, handshake

*** Test Framework API
- =define-hegel-test= macro for registering property tests
- =run-hegel-tests!= for executing all registered tests against a single server connection
- =tc-draw= for requesting server-generated values
- =tc-assume= for precondition rejection (raises exception on =#f=)

*** Generator Schemas
- =integers= --- =#:min-value=, =#:max-value=
- =booleans= --- =#:p=
- =floats= --- =#:min-value=, =#:max-value=, =#:allow-nan=, =#:allow-infinity=
- =text= --- =#:min-size=, =#:max-size=
- =binary= --- =#:min-size=, =#:max-size=
- =null-values=
- =lists-of= --- =#:min-size=, =#:max-size=, =#:unique=
- =one-of= --- variadic schemas
- =sampled-from= --- list of items
- =const-value= --- single value
- =gen-filter= --- client-side predicate filtering via =tc-assume=
- =gen-map= --- client-side transform

*** Examples

** Changed

These are protocol corrections discovered during development. They represent
deviations from the initial spec assumptions in =CLAUDE.md=, resolved through
live probing and CPRR.

- Framing: HEGL packet format (magic + CRC32 + channels) replaces the initial assumption of bare uint32-BE length-prefixed CBOR frames
- Version strategy: version number tracks Hegel protocol version (=0.7.x= for protocol 0.7)
- Status strings: UPPERCASE (=VALID= / =INVALID= / =INTERESTING=) matching Python =Status= enum, not lowercase (=passed= / =failed= / =invalid=)
- Schema type names: singular forms (=integer= not =integers=, =string= not =text=)
- Reply format: all replies wrapped in ={"result": value}= envelope (C10)
- =run_test= message: =channel_id= required at top level (C12)
- Server discovery: client creates socket path, passes as CLI arg (not read from server stdout)
- Test case lifecycle: server-driven (server sends =test_case= events), not client-driven (client no longer sends =start_test_case=)
